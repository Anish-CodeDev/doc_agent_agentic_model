# doc\_agent\_agentic\_model

A fine-tuned language model that powers agentic, multi-step reasoning for the `doc_agent` component of a Desktop Agent and `system_agent` MCP Server. Given a natural-language task involving documents or presentations, the model generates a structured, step-by-step reasoning chain that maps directly to tool calls.

***

## Overview

This project fine-tunes **Qwen2.5-3B-Instruct** using **Unsloth + LoRA (SFT)** on a synthetically generated dataset of document/presentation manipulation tasks. The trained model is then served via a lightweight **Flask API**.

The model is designed to:

* Parse a high-level user instruction (e.g., *"Rewrite the Safety Protocols section in contract.docx and then set font to Arial"*)

* Emit a structured chain-of-thought reasoning trace wrapped in `<think>...</think>` tags

* Map each reasoning step to a concrete tool call (`modify_text_of_doc`, `find_slide_tool`, etc.)

## Dataset

The training data is synthetically generated by `data/gen_data.py`. It produces 5,000 multi-step task samples across four action types:

| Task Type           | Example Input                                                                                               |
| ------------------- | ----------------------------------------------------------------------------------------------------------- |
| Delete slide        | "Find the slide about AI Ethics in pitch\_deck.pptx and delete it."                                          |
| Modify presentation | "Locate the Budget slide in quarterly\_review.slides and update its content."                                |
| Update document     | "Rewrite the Q3 Revenue section in report\_2026.pdf and then bold all headers."                              |
| Insert slide        | "I need a new slide for Safety Protocols at the start of final\_presentation.pptx, then update its content." |

Each sample contains:

* **`Input_Task`** — natural language instruction

* **`Reasoning_Prompts`** — ordered tool call steps as ground truth output

## Fine-Tuning

The notebook `model_finetuning.ipynb` covers the full training pipeline:

**Base model:** `unsloth/Qwen2.5-3B-Instruct`\
**Method:** LoRA via Unsloth (4-bit quantized)\
**Sequence length:** 512 tokens\
**LoRA rank:** 16\
**LoRA targets:** `q_proj`, `k_proj`, `v_proj`, `o_proj`, `gate_proj`, `up_proj`, `down_proj`

**SFT config highlights:**

* Optimizer: `adamw_8bit`

* Learning rate: `2e-4`

* Epochs: 2

* Batch size: 2 (with gradient accumulation steps: 2)

After training, the model is saved as `sft_model/` and tokenizer as `sft_tokenizer/`.

## Inference Server

`server.py` exposes the fine-tuned model as a Flask API.

### Run the server

```
pip install flask unsloth transformers
python server.py
```

> Requires a CUDA-enabled GPU. The model loads in 4-bit quantized mode.

### API

**Endpoint:** `POST /`

**Request body:**

```
{
  "request": "Rewrite the Safety Protocols section in contract.docx and then set font to Arial."
}
```

**Response:**

```
<think>
1. Use modify_text_of_doc(filename='contract.docx', topic='Safety Protocols').
2. Use modify_formatting_of_doc(filename='contract.docx', request='set font to Arial').
</think>
```

The response contains the model's reasoning trace which the calling agent (`doc_agent` / `system_agent`) parses to execute the appropriate tool calls.
